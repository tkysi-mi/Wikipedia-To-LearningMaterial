手動での教材作成には多大な時間と労力が必要であり、特に社会人が新しいトピックを学習する際、Wikipedia記事を受動的に読むだけでは注意力が低下しやすく、学習効果が限定的である。また、AI教材生成の実用性を検証するためのデモアプリケーションが不足している。

Wikipedia記事のURLを入力すると、AIが自動的に3行のサマリと10問の○×問題を生成し、問題を1問ずつ解くことで記事を注意深く読むモチベーションを向上させる。これにより、受動的な読書から能動的な問題解決型の学習へ転換し、知識の定着を促進する。

### 含まれる機能（MVP） - ✅ 実装完了（2025-11-22）

- **Wikipedia URL入力**: Wikipedia記事のURLを入力して教材を生成
- **AIによるサマリ生成**: OpenAI APIで記事を3行程度に要約
- **AIによる○×問題生成**: OpenAI APIで記事から10問の○×問題を生成
- **問題回答UI**: 1問ずつ表示し、○/×ボタンで回答
- **正解/不正解フィードバック**: 回答後に即座に正誤を表示
- **結果表示**: 全問終了後に正答数・正答率を表示
- **ベーシック認証**: 共通パスワードで不正アクセスを防止
- **エラーハンドリング**: 無効URL、Wikipedia API失敗、OpenAI API失敗時のエラーメッセージ表示
- **ローディング表示**: OpenAI API呼び出し中のローディングスピナー

**実装タスク**: [task000003-implement-core-features](file:///c:/Users/Takey/Desktop/Wikipedia-To-LearningMaterial/docs/tasks/task000003-implement-core-features)

### 意図的に含めない機能（デモ用途のため）

- **ユーザー管理**: 個別アカウント不要、共通パスワードのみ
- **学習履歴の永続化**: データベース不要、インメモリのみ（サーバー再起動でデータ消失）
- **問題の再現性**: 同じ記事でも問題は毎回異なる（OpenAI APIの特性）
- **難易度調整**: 問題の難易度選択機能なし
- **多言語UI**: UI表示は日本語のみ（Wikipedia記事は日本語・英語両対応）

### 将来的に検討する機能（Phase 2候補）

- **データベース導入**: PostgreSQLで学習履歴を永続化
- **ユーザー登録とログイン**: 個別アカウントで学習履歴を管理
- **学習統計ダッシュボード**: 累計学習記事数、平均正答率、学習時間の可視化
- **ストリーミング表示**: OpenAI APIストリーミングレスポンスでUX向上
- **問題形式の多様化**: 多肢選択問題、記述式問題など
- **難易度調整**: 問題の難易度（初級・中級・上級）選択機能
- **他の知識ソース**: ニュース記事、学術論文などへの対応
- **モバイルアプリ対応**: React NativeでiOS/Androidアプリ開発

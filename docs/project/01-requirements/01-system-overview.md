手動での教材作成には多大な時間と労力が必要であり、特に社会人が新しいトピックを学習する際、Wikipedia記事を受動的に読むだけでは注意力が低下しやすく、学習効果が限定的である。また、AI教材生成の実用性を検証するためのデモアプリケーションが不足している。

Wikipedia記事のURLを入力すると、AIが自動的に3行のサマリと10問の○×問題を生成し、問題を1問ずつ解くことで記事を注意深く読むモチベーションを向上させる。これにより、受動的な読書から能動的な問題解決型の学習へ転換し、知識の定着を促進する。

社会人（自習者）を主要ユーザーとし、基本的なWebブラウザ操作ができる初心者〜中級者を想定。デモ用途のため、同時利用は2-3人程度。PC、スマートフォン、タブレットのモダンブラウザに対応。

### 含まれる機能（MVP）
- Wikipedia URL入力
- AIによるサマリ生成（3行程度）
- AIによる○×問題生成（10問）
- 問題回答UI（1問ずつ表示）
- 正解/不正解フィードバック
- 結果表示（正答数・正答率）
- ベーシック認証（共通パスワード）
- エラーハンドリング（無効URL、API失敗時）

### 意図的に含めない機能
- ユーザー管理（個別アカウント不要、共通パスワードのみ）
- 学習履歴の永続化（データベース不要、インメモリのみ）
- 問題の再現性（同じ記事でも問題は毎回異なる）
- 難易度調整
- マイクロインタラクション（ローディングアニメーションなど）
- 多言語対応（日本語のみ）

### 将来的に検討する機能
- ユーザー登録とログイン
- 学習履歴のトラッキング
- 問題の難易度選択
- 他の知識ソース（ニュース記事、学術論文など）への対応
- 問題形式の多様化（選択式、記述式など）

| カテゴリ | 要件 | 説明 |
|----------|------|------|
| 性能 | 静的ページ応答時間 | URL入力画面などの静的ページは1秒以内に表示 |
| 性能 | サマリ生成時間 | 10-30秒程度を許容（OpenAI API呼び出し含む） |
| 性能 | 問題生成時間 | 30-60秒程度を許容（OpenAI API呼び出し含む） |
| 性能 | 問題回答応答時間 | ○/×選択後のフィードバックは1秒以内 |
| 性能 | 同時接続数 | 2-3人の同時利用を想定、最大5人程度まで対応 |
| セキュリティ | 認証方式 | ベーシック認証（共通パスワード1つ） |
| セキュリティ | パスワード管理 | 環境変数で管理（ハードコード禁止） |
| セキュリティ | APIキー管理 | OpenAI APIキーは環境変数に保存、Gitにコミットしない |
| セキュリティ | HTTPS | Railway提供のHTTPSを使用（本番環境） |
| セキュリティ | データ保護 | 個人情報を収集しない、学習履歴を保存しない |
| 可用性 | 稼働率 | 95%以上（デモ用途のため、高可用性は不要） |
| 可用性 | 許容ダウンタイム | 月間36時間程度まで許容 |
| 可用性 | 復旧時間 | Railway で再デプロイ、1時間以内 |
| スケーラビリティ | 初期ユーザー数 | 2-3人同時利用 |
| スケーラビリティ | データ量 | インメモリデータのみ（セッションごとに1-5KB程度） |
| スケーラビリティ | スケール方針 | 垂直スケール（サーバースペック向上）でOK |
| ユーザビリティ | レスポンシブデザイン | PC、スマートフォン、タブレット対応 |
| ユーザビリティ | 対応ブラウザ | Chrome、Firefox、Safari、Edge の最新版 |
| ユーザビリティ | アクセシビリティ | WCAG 2.1 Level A を目指す |
| ユーザビリティ | 多言語対応 | 日本語のみ（日本語版Wikipedia想定） |
| 保守性 | ログ | 基本的なエラーログを標準出力に記録 |
| 保守性 | 監視 | Railway のログ機能で十分（高度な監視不要） |
| 保守性 | デプロイ | Git push で自動デプロイ（Railway） |
| 保守性 | ロールバック | Railway のデプロイ履歴から10分以内に復元 |
| 互換性 | Wikipedia API | 公開API、特定バージョンに依存しない |
| 互換性 | OpenAI API | GPT-4.5 mini または類似モデル |

## 補足

本プロジェクトは**デモ・実証実験**のため、以下の点を重視します：

- **シンプルさ**: 複雑な構成を避け、最小限の構成で実装
- **低コスト**: インフラ・API費用を抑える
- **迅速な開発**: MVP を短期間で完成させる
- **実用性**: 基本的な動作が確実に行える

高度なパフォーマンス、セキュリティ、可用性は求めず、基本的な要件を満たすことを目標とします。
